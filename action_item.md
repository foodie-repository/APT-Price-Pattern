## 2. 가장 중요한 구조 보강

### 2.1 단지 식별자 표준화

목표:
- 같은 단지를 시점별, 데이터셋별로 안정적으로 이어 붙일 수 있게 만들기

핵심 작업:
- [ ] `단지고유번호`, `단지코드`, `주소 정규화`, `단지명 정규화` 체계 정리
- [ ] 재건축/재개발 전후 `단지명 이력` 정리
- [ ] 분할/통합, 브랜드 변경 이력 추적 규칙 정리

이유:
- 단지 단위 분석
- 정비사업 추적
- 과거와 현재 비교
- 서로 다른 데이터 소스 조인 안정화

### 2.2 전국 공통 변수 vs 수도권 심화 변수 분리

전국 공통:
- 연식
- 세대수
- 동수
- 면적 구성
- 거래량
- 가격 수준
- 급지/상대서열
- 임차
- 정책
- 공급

수도권 심화:
- 역세권
- 학교/학원
- 상권
- 종합병원/대규모점포
- 공원
- 한강
- CBD
- 정비사업

핵심 작업:
- [x] 문서 기준에서 전국 공통 변수와 수도권 심화 변수 구분
- [x] DuckDB 전처리/마스터 뷰 단계에서 실제 테이블 분리
- [ ] 보고서에서 두 범위를 섞지 않도록 산출 구조 분리

이유:
- 수도권 분석 구조를 지방에 그대로 복제하면 왜곡이 커짐

### 2.3 예측 대상 정의

권장:
- 기본 예측 단위: `읍면동`
- 기본 예측 기간: `향후 6개월`
- 출력: `상승확률`, `가격 반영이 덜 된 가능성 점수`, `변화 계기 점수`, `수익률 후보 점수`
- 단지 단위 예측은 표본 기준 충족 시만 수행

핵심 작업:
- [x] 문서 기준에서 예측 대상 정의 반영
- [x] 실제 예측 파이프라인과 점수 산식 구현

### 2.4 투자 검토 대상군 및 실제 검토 적합도 보정

목표:
- `상승확률`과 `투자 가치`를 같은 개념으로 취급하지 않기

핵심 작업:
- [x] 투자 검토 대상군 및 실제 검토 적합도 보정 설계 문서 작성
- [x] 사람 검증셋 초기본 작성
- [ ] `방향성 엔진 + 투자 판단 엔진` 2단 구조로 예측 파이프라인 명확화
- [x] `즉시 제외 / 판단 보류 / 우선 검토` 규칙을 실제 리포트와 점수 산식에 더 정교하게 반영

이유:
- 통계적으로는 맞아 보여도 투자자 관점에서 납득되지 않는 후보를 줄이기 위해

## 3. 추가로 수집하면 가장 가치 큰 데이터

### 3.1 필수

- [ ] 전국 단지 식별/단지명 이력 데이터
- [ ] 전국 단지 속성 데이터
- [ ] 전국 단지 좌표
- [ ] 공급 정식 데이터

세부 예시:
- 단지 식별: 한국부동산원 공동주택 단지 식별정보, 공동주택관리정보시스템 계열
- 단지 속성: 주차대수, 최고층, 복도유형, 난방방식, 시공사/브랜드
- 공급: 인허가, 착공, 분양, 입주, 준공, 미분양, 멸실

### 3.2 권장

- [ ] 학군의 질 데이터
- [ ] 현재 매물/호가/회전 데이터
- [ ] 지역 수요/배후 데이터

세부 예시:
- 학군: 학업성취, 특목고/자사고 진학, 선호 학군 경계
- 매물/호가: 네이버/호갱노노 계열의 매물 수, 증감, 체류일수, 호가 흐름
- 수요/배후: 가구수, 연령대, 순이동, 소득, 고용, 산업단지 종사자, 직주근접 축

### 3.3 있으면 좋지만 우선순위는 낮음

- [ ] 학군 경계 폴리곤
- [ ] 정비사업 단계별 공간 경계
- [ ] 도로망 기반 실제 이동시간 데이터
- [ ] 조망권/수변/고저차 데이터
- [ ] 토지이용계획, 용도지역, 용적률/건폐율
- [ ] 법원경매/공매 데이터
- [ ] 분양권/입주권 거래 데이터

## 4. 분석 설계상 추가 검토 포인트

### 4.1 서울/수도권

- [x] 노후 단지는 단순 감가상각으로 해석하지 않는다는 원칙을 문서에 반영
- [ ] `재건축 기대`, `토지 희소성`, `한강/CBD 접근성`, `핵심 생활권 독점성`을 실제 분석 변수와 해석에 반영

### 4.2 지방광역시/주요중소도시

- [x] `역세권`보다 `자차 생활권`, `산업`, `행정`, `도심`, `신도심`, `IC 접근성` 축을 우선 해석한다는 원칙을 문서에 반영
- [ ] 지방 분석 결과에서 수도권 변수 체계를 실제로 분리 적용

### 4.3 투자 관점 지표 구분

- [x] `가격 반영이 덜 된 상태`, `상승확률`, `수익률 후보`를 구분하는 원칙을 문서에 반영
- [ ] 실제 스코어링과 보고서에서 3개 지표를 분리 산출

정의:
- `가격 반영이 덜 된 상태`: 현재 조건 대비 덜 반영된 상태
- `상승확률`: 향후 6개월~12개월에 실제 움직일 가능성
- `수익률 후보`: 전세가율, 월세흐름, 투자금 부담까지 고려한 후보

### 4.4 단지 단위 분석 시 주의
