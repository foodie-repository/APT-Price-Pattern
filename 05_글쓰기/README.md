# Writing Workspace

이 폴더는 동일한 분석 사실을 바탕으로, 채널별로 다른 문체와 편집 구조를 적용한 발행용 원고를 관리하는 공간이다.

구조:

- `05_글쓰기/01_네이버블로그/`
- `05_글쓰기/02_네이버프리미엄컨텐츠/`
- `05_글쓰기/03_브런치스토리/`

공통 원칙:

- 세 채널 원고는 같은 분석 결과와 같은 사실 기반을 공유한다.
- 채널별 차이는 `톤`, `서술 밀도`, `독자 가정`, `문단 구성`, `제목 전략`에서만 둔다.
- 원고 안의 핵심 사실, 날짜, 수치, 정책 해석은 보고서와 모순되면 안 된다.
- 모든 결론은 가능하면 `사실 / 해석 / 가설` 구조를 유지한다.
- 과장된 확정 표현은 피하고, 투자 판단은 조건과 한계를 함께 적는다.

채널별 기본 방향:

- `네이버블로그`: 검색 친화적, 설명형, 실용적, 문단 짧게
- `네이버프리미엄컨텐츠`: 유료 구독형 톤, 구조적, 밀도 높게, 투자자 관점 강하게
  - 이론적 설명, 방법론 맥락, 실제 사례, 한계와 적용 조건까지 포함한다.
  - 무료 공개형 글보다 `왜 그렇게 해석하는가`를 더 길고 구체적으로 설명한다.
  - 독자가 다음 글에서도 재사용할 수 있는 판단 프레임을 남기는 것을 목표로 한다.
- `브런치스토리`: 에세이형, 서사형, 문제의식과 관찰 흐름 강조

파일명 규칙:

- `{순번}_{주제}_{날짜}_{채널}.md`
- 예: `00_통합분석방법론_20260313_네이버블로그.md`
