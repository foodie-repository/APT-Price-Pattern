# 추가 지표 도입 로드맵

작성일: 2026-03-15

## 목적
- 이 문서는 현재 프로젝트의 분석력과 예측력을 높이기 위해 어떤 추가 지표를 어떤 순서로 붙일지 정리한 설계 문서다.
- 핵심은 `데이터를 많이 모으는 것`이 아니라, `이미 강한 가격/거래/임차/정책/공급 체계 위에 어떤 레이어를 얹어야 판단력이 실제로 좋아지는가`를 분명히 하는 데 있다.
- 실제 원천 수집은 별도 프로젝트 `Real-Estate-Data`에서 진행하고, 이 문서는 `APT-Price-Pattern`에서의 반영 기준과 역할을 정의한다.

## 현재 상태 요약
- 현재 DuckDB에 실제로 무엇이 들어와 있는지는 `02_데이터/04_메모/DuckDB_보유데이터_인벤토리_20260315.md`를 기준으로 확인한다.
- 현재 프로젝트는 `가격`, `거래`, `임차`, `정책`, `입주예정 물량`, `미분양`, `준공후 미분양`, `상품성/입지`, `실제 검토 적합도`, `과거 데이터 검증`의 뼈대가 이미 구축되어 있다.
- `KOSIS_아파트매매_거래규모별`, `KOSIS_아파트매매_거래주체별`, `KOSIS_아파트매매_매입자거주지별`, `KOSIS_아파트매매_매입자연령대별`은 이미 DuckDB에 반영된 상태다.
- `주택도시보증공사(HUG)` 분양 관련 지표도 DuckDB에 반영된 상태다.
  - `KOSIS_HUG_분양가격`
  - `KOSIS_HUG_분양가격지수`
  - `KOSIS_HUG_분양세대수`
  - `KOSIS_HUG_초기분양률`
  - `KOSIS_HUG_대지비비율`
- `KOSIS` 부동산 관련 소비심리지수도 DuckDB에 반영된 상태다.
  - `KOSIS_소비심리_부동산시장`
  - `KOSIS_소비심리_주택시장`
  - `KOSIS_소비심리_주택매매`
  - `KOSIS_소비심리_주택전세`
  - `KOSIS_소비심리_토지시장`
- `KB` 민간 심리지수도 DuckDB에 반영된 상태다.
  - `KB_매수우위지수`
  - `KB_매매가격전망지수`
  - `KB_매매거래활발지수`
- 따라서 추가 작업의 핵심은 `거래구성 4종`, `HUG 분양시장 지표`, `KOSIS 공식 심리`, `KB 민간 심리`를 더 잘 쓰는 것과 `금융조건`, `뉴스`, `구조적 리스크` 레이어를 순차적으로 보강하는 것이다.

## 추가 지표를 붙이는 6개 레이어

### 1. 핵심 뼈대 레이어
- 가격 변화율
- 거래 회복
- 임차 시장의 지지력
- 정책/규제
- 입주예정 물량, 미분양, 준공후 미분양
- 상품성/입지

설명:
- 이 레이어는 이미 현재 프로젝트의 중심축이다.
- 추가 지표는 이 뼈대를 대체하는 것이 아니라, 해석을 더 정교하게 하거나 전환점을 더 빨리 감지하는 역할을 해야 한다.

### 2. 거래구성 레이어
- 거래규모별
- 거래주체별
- 매입자거주지별
- 매입자연령대별

설명:
- `거래가 늘었다`는 사실을 넘어 `누가`, `어떤 규모를`, `어디에서 들어와서` 사고 있는지를 해석하게 해 준다.
- 이 레이어는 이미 데이터가 들어와 있으므로 신규 수집보다 활용 고도화가 우선이다.

### 3. 1차 분양시장 레이어
- 민간아파트 분양가격
- 초기분양률
- 신규 분양세대수

설명:
- 기존 주택시장보다 한 박자 먼저 움직일 수 있는 신호다.
- 특히 `회복 초입`, `과열 초입`, `정책 완화 효과의 선행 반영`을 읽는 데 유용하다.

### 4. 금융조건 레이어
- 기준금리
- 주담대금리
- 고정/변동금리 비중
- 가계대출
- 대출수요/대출태도

설명:
- 같은 가격 조정기라도 금리 2%와 5%는 전혀 다른 시장이다.
- 금융조건은 지역별 횡단면 비교 변수라기보다 `전국 공통 국면 변수`로 다루는 것이 맞다.
- 현재 DuckDB에는 `KOSIS_BOK_대출금리_신규`, `KOSIS_BOK_대출금리_잔액`, `KOSIS_BOK_가계대출_업권별`, `KOSIS_BOK_가계대출_용도별`, `KOSIS_BOK_가계대출_지역별`, `KOSIS_BOK_대출수요`, `KOSIS_BOK_대출태도`가 반영되어 있다.
- 현재 버전에서는 `시군구 직접 점수`보다 `장세 배경`, `정책 효과 상쇄`, `시나리오 촉발 조건` 해석에 먼저 사용한다.

### 5. 심리/관심 레이어
- 공식 심리 지표
- 뉴스 기사량
- 뉴스 주제 태그
- 검색 관심 급증

설명:
- 가격과 거래를 대체하는 축이 아니라 `전환점 탐지`, `과열 경고`, `데이터와 서사의 괴리 확인`을 위한 보조축이다.
- 감성분석을 먼저 넣기보다 `기사량 급증`, `주제 분류`, `뉴스-거래 괴리`부터 시작하는 편이 안정적이다.

공식 심리 내부 구분:
- 현재 프로젝트에서 `공식 심리`의 1순위는 이미 수집한 `KOSIS_소비심리_부동산시장`, `KOSIS_소비심리_주택시장`, `KOSIS_소비심리_주택매매`, `KOSIS_소비심리_주택전세`다.
- 이 지표들은 현재 프로젝트에 가장 직접적으로 맞닿아 있는 `부동산 직접 심리`다.
- 그 다음 추가 검토 후보는 한국은행 계열의 `주택가격전망CSI`, `소비자심리지수(CCSI)`, `가계부채CSI`, `건설업 BSI`, `금리수준전망CSI`, `경제심리지수(ESI)`다.
- 위 후보들은 현재 지표를 대체하는 것이 아니라 `교차검증`과 `전국 공통 국면 확인`을 위한 보강축으로 보는 편이 맞다.

### 6. 구조적 보정 레이어
- 인구이동
- 주택소유통계
- 가계금융복지조사
- 민간임대주택 재고/공급
- 빈집 통계
- 정책/기업 투자 이벤트 DB

설명:
- 단기 방향을 직접 맞히는 신호라기보다 `투자 검토 게이트`, `실제 검토 적합도`, `구조적 설명력`을 보강하는 층이다.
- 특히 정부·기업 투자 방향은 하나의 월별 숫자보다 `지역 이벤트 DB`로 관리하는 편이 더 실무적이다.

## 역할 분류 원칙

### 1. 국면 변수
- 전국 공통으로 시장의 배경을 바꾸는 변수다.
- 예: 기준금리, 주담대금리, 가계대출, 대출수요/대출태도, 공식 심리 지표

### 2. 선행 신호
- 기존 주택시장보다 조금 먼저 변하는 가능성이 있는 변수다.
- 예: 초기분양률, 신규 분양세대수, 뉴스 기사량 급증

### 3. 거래구성 신호
- 거래 회복의 질을 분해해 주는 변수다.
- 예: 매입자거주지별, 매입자연령대별, 거래규모별

### 4. 지역 보정 변수
- 같은 가격 흐름이라도 지역별 질이 다른 이유를 보완하는 변수다.
- 예: 인구이동, 지역 이벤트 DB, 민간임대주택 재고

### 5. 실제 검토 적합도 보정
- 방향성은 있어도 투자 검토 우선순위는 낮춰야 하는 지역을 걸러내는 변수다.
- 예: 빈집, 장기 미분양 성격, 소유집중, 출구 취약성 관련 통계

### 6. 설명력 보강 변수
- 예측보다 보고서의 납득력과 맥락 설명을 강화하는 변수다.
- 예: 가계금융복지조사, 주택소유통계

## 우선순위와 도입 순서

### 1차 도입
- 민간아파트 분양가격
- 초기분양률
- 신규 분양세대수
- 기준금리
- 주담대금리
- 가계대출
- 대출수요/대출태도
- 기존 거래구성 4종의 활용 고도화

목표:
- `회복 초입과 과열 초입`을 더 빨리 읽는다.
- `돈의 가격과 레버리지 비용`을 현재 국면 설명에 정식으로 포함한다.
- `누가 사고 있는가`를 매매시장 해석과 예측 점수에 더 명시적으로 반영한다.
- 다만 `금융조건`, `심리`, `분양시장`은 현재 예측/검증 단계에서 직접 점수화보다 `국면 보정`이 우선이다.

우선 붙일 이유:
- 분양시장과 금융조건은 기존 뼈대와 결합했을 때 설명력 증가가 가장 크다.
- 매입자거주지별, 매입자연령대별은 이미 들어와 있으므로 즉시 활용 효과를 볼 수 있다.
- HUG 분양 지표는 이미 DuckDB에 들어왔으므로, 다음 단계는 `파생지표 정의 -> 매매/정책/예측 시범 반영`이다.

### 2차 도입
- 인구이동
- 공식 심리 지표
- 뉴스 기사량과 주제 태그
- 정책/기업 투자 이벤트 DB

목표:
- `실수요 유입`, `심리 변화`, `지역별 변화 계기`를 보강한다.
- 정책·개발·산업·교통 이슈를 숫자 한 줄이 아니라 지역 이벤트 레이어로 붙인다.

우선순위를 2차로 둔 이유:
- 인구이동과 심리는 중요하지만, 1차 시장과 금융조건보다 직접적인 즉시효과는 조금 약하다.
- 뉴스는 수집보다 활용 설계가 더 중요해서, 분류 기준을 먼저 정해야 한다.
- 공식 심리 지표는 이미 DuckDB에 들어왔으므로, 다음 단계는 `뉴스와 역할 분리`, `심리-거래 괴리`, `심리-전세 괴리`를 정식 해석축으로 고도화하는 일이다.

공식 심리 추가 후보 우선순위:
1. `주택가격전망CSI`
2. `소비자심리지수(CCSI)`
3. `가계부채CSI`
4. `건설업 BSI`
5. `금리수준전망CSI`
6. `경제심리지수(ESI)`

설명:
- `주택가격전망CSI`는 현재 보유한 `주택매매 소비심리`와 출처가 달라 교차검증 가치가 크다.
- `CCSI`와 `가계부채CSI`는 지역 선별 지표라기보다 `전국 공통 국면 변수`로 쓸 가능성이 높다.
- `건설업 BSI`는 의미는 있지만, 이미 `HUG 분양가격/초기분양률/분양세대수`가 들어와 있는 현재 프로젝트에서는 우선순위를 조금 낮춰도 된다.
- `ESI`는 `CCSI`와 `BSI`를 별도로 볼 경우 중복 효과가 클 수 있어 후순위가 적절하다.

### 3차 도입
- 주택소유통계
- 가계금융복지조사
- 민간임대주택 재고/공급
- 빈집 통계

목표:
- 지방 리스크와 구조적 취약성을 더 잘 구분한다.
- `실제 검토 적합도`와 `회피/관찰 유지` 판정을 더 단단하게 만든다.

우선순위를 3차로 둔 이유:
- 대부분 연간 또는 저빈도 구조 통계라 단기 타이밍 신호보다는 보정 변수 역할에 가깝다.
- 방향성 엔진보다 투자 판단 엔진 보강에 더 적합하다.

## 권장 도입 절차

### 1. 외부 수집 완료 확인
- `Real-Estate-Data`에서 원천 수집과 기본 정리를 마친다.
- 표 ID, 기준월, 지역 단위, 결측 구조를 먼저 확인한다.

### 2. 반입 전 메타 점검
- 이 프로젝트에서 사용할 최소 컬럼을 확정한다.
- 지역 코드와 지역명 체계가 현재 DuckDB와 연결 가능한지 확인한다.
- 최신 기준월과 이력 범위를 기록한다.

### 3. DuckDB 적재
- 원천 테이블을 그대로 넣는 층과, 분석용 집계 뷰를 나눠서 관리한다.
- 원천과 파생 뷰의 역할을 섞지 않는다.

### 4. 파생지표 정의
- 단순 수준값보다 `전년동기`, `최근 3개월`, `최근 6개월`, `이동평균`, `급증 배수`를 우선 고려한다.
- 금리처럼 전국 공통 축은 `국면 분류 변수`로, 지역 자료는 `횡단면 보정 변수`로 분리한다.

### 5. 시범 반영
- 먼저 `매매시장 분석`, `정책영향분석`, `예측/검증`에 제한적으로 넣어 본다.
- 한 번에 모든 보고서에 전면 반영하지 않는다.

### 6. 점수/게이트 연결
- `상승확률 점수`
- `가격 반영이 덜 된 가능성 점수`
- `변화 계기 점수`
- `가격이 앞서간 가능성 점수`
- `실제 검토 적합도 게이트`

위 축 중 어디에 보조 입력으로 넣을지 명확히 정한 뒤에만 점수화한다.

### 7. 과거 데이터 검증
- 새 지표를 넣기 전과 후의 성능을 비교한다.
- 상위 후보 품질, 제외 후보 품질, 사람 검증셋 납득도까지 함께 본다.

### 8. 보고서 문장 정리
- `새 지표가 무엇을 추가로 설명했는가`
- `기존 해석을 어떻게 바꿨는가`
- `여전히 보조축일 뿐인 이유는 무엇인가`

위 세 문장을 각 보고서에 명시한다.

## 세부 도입 원칙

### 분양시장 지표
- 분양가격은 기존 주택가격과 직접 치환하지 않고 `괴리`와 `프리미엄`으로 해석한다.
- 초기분양률은 단독 강세 신호가 아니라 거래, 전세가율, 정책 방향과 함께 본다.
- 신규 분양세대수는 공급 파이프라인이 아니라 `1차 시장 온도`의 보조축으로 본다.

### 금융조건 지표
- 기준금리는 국면 전환의 배경 변수다.
- 주담대금리는 실제 수요 부담에 더 가깝기 때문에 우선 활용도가 높다.
- 가계대출은 절대수준보다 증가율, 둔화율, 레버리지 확장/축소 방향이 중요하다.
- 대출수요와 대출태도는 실제 금리와 가계대출을 보완하는 금융환경 온도 지표다.
- 예측/검증 단계에서는 우선 `상승/보합/하락 시나리오`, `후보 해석`, `정책 효과의 상쇄 여부` 설명에 넣고, 별도 검증 없이 시군구 직접 점수에 기계적으로 더하지 않는다.

### 뉴스와 심리 지표
- 뉴스는 처음부터 방향성 점수의 핵심 입력으로 넣지 않는다.
- 1단계는 `기사량 급증`, 2단계는 `주제 분류`, 3단계는 `뉴스-거래 괴리`다.
- 공식 심리 지표와 뉴스 지표는 역할을 분리한다.
  - 공식 심리: 전국 또는 권역 단위의 공식 기대 변화
  - 뉴스: 지역별 관심 급증과 서사 변화
- 공식 심리 추가 후보를 붙일 때도 지역 랭킹 지표로 바로 쓰지 말고, 우선 `현재 보유한 부동산 직접 심리`와의 괴리와 시차를 확인한다.

### 정책/기업 투자 이벤트 DB
- 교통, 산업, 규제완화, 정비사업, 공공투자, 기업투자는 단일 지수보다 `이벤트 날짜 + 지역 + 강도 + 유형` 구조가 낫다.
- 이 레이어는 정책영향분석과 변화 계기 점수 보조축에 우선 연결한다.

## 성공 기준
- 추가 지표가 들어간 뒤 `현재 국면 설명`이 더 구체적이고 방어 가능해진다.
- 상위 후보의 납득도가 높아지고, 사람이 봤을 때 이상한 후보가 줄어든다.
- `왜 이번에는 이 지역을 우선 검토/관찰 유지/보수 접근/회피로 보는가`를 더 선명하게 설명할 수 있다.
- 과거 데이터 검증에서 적어도 일부 버킷의 품질 개선이나 해석 일관성 개선이 확인된다.

## 이번 단계의 실무 결론
- `민간아파트 분양가격 / 분양가격지수 / 신규 분양세대수 / 초기분양률 / 대지비비율`은 `HUG` 기준으로 이미 DuckDB에 반영되었다.
- `KOSIS_소비심리_부동산시장 / 주택시장 / 주택매매 / 주택전세 / 토지시장`도 이미 DuckDB에 반영되었다.
- `KB_매수우위지수 / KB_매매가격전망지수 / KB_매매거래활발지수`도 이미 DuckDB에 반영되었다.
- `KOSIS_ECOS_기준금리 / KOSIS_BOK_대출금리_신규 / KOSIS_BOK_대출금리_잔액 / KOSIS_BOK_가계대출_업권별 / KOSIS_BOK_가계대출_용도별 / KOSIS_BOK_가계대출_지역별 / KOSIS_BOK_대출수요 / KOSIS_BOK_대출태도`도 이미 DuckDB에 반영되었다.
- 금융조건 축의 다음 작업은 `최신 기준월 확인 -> 파생지표 정의 -> 매매/임차/정책/예측 시범 반영`이다.
- 필요 시 `고정/변동금리 비중`만 추가 보강 후보로 유지한다.
- `매입자거주지별`, `매입자연령대별`은 이미 들어와 있으므로 신규 수집이 아니라 `반영 고도화` 대상으로 본다.
- 심리 지표 측면에서는 `이미 보유한 KOSIS 소비심리지수`가 우선이고, `주택가격전망CSI`, `CCSI`, `가계부채CSI`, `건설업 BSI`, `금리수준전망CSI`, `ESI`는 그 다음에 붙일 추가 공식 심리 후보로 본다.
- 정부와 기업의 투자 방향은 별도 월별 숫자 하나를 찾기보다 `정책/기업 투자 이벤트 DB`로 병행하는 편이 낫다.
- 뉴스는 감성점수보다 `기사량`, `주제`, `데이터와의 괴리`부터 쓰는 편이 맞다.
- 현재 기준에서 가장 가까운 다음 작업은 `HUG 분양지표`와 `KOSIS 소비심리지수`를 매매시장 분석, 정책영향분석, 예측/검증에 시범 반영하는 일이다.
