# 상품성/입지 데이터 카탈로그

이 문서는 `/Users/foodie/myproject/APT-토지가치분석`에서 재사용 가능한 상품성/입지 데이터를 현재 프로젝트의 `02_데이터` 구조로 가져온 뒤 정리한 카탈로그다.

중요 원칙:
- 이 폴더는 `이전 프로젝트의 결론`을 가져오는 곳이 아니라 `데이터와 변수 후보`를 재사용하는 곳이다.
- 현재 프로젝트의 최종 해석은 항상 `실거래 시계열 + 임차 + 정책 + 공급 + 상품성/입지`를 함께 보고 새로 만든다.
- 수도권 심화 분석과 지방 보조 분석을 분리한다.

## 핵심 활용 원칙

1. 좌표는 DuckDB 내부 소스를 우선 사용한다.
   - 1순위: DuckDB 내부 좌표 소스 (`main.좌표`, `공동주택_서울.좌표X/좌표Y` 등)
   - 2순위: `02_데이터/03_가공/상품성/단지_좌표.csv`
   - 항상 `좌표출처`, `매칭키`, `매칭성공률`을 남긴다.

2. 수도권 심화 변수와 전국 공통 변수는 분리한다.
   - `02_데이터/03_가공/상품성/단지_좌표.csv`와 `02_데이터/03_가공/상품성/아파트_단지_위치변수.csv`는 서울/경기/인천 `19,326개 단지` 기준이다.
   - 전국 공통 분석에는 그대로 일반화하지 않는다.
   - 전국 분석의 기본 축은 `연식`, `세대수`, `동수`, `거래량`, `급지/상대서열`, `가격 수준`, `임차`, `정책`, `공급`이다.
   - 수도권 심화 분석에서만 `역세권`, `학원밀집도`, `상권`, `공원`, `한강`, `CBD`, `정비사업`을 적극 사용한다.

3. `단지_메타정보.csv`는 보조 자료로 쓴다.
   - 거래/공시가격 집계 기반 메타정보라서, 현재 프로젝트의 `정식 단지 마스터`를 대체하지 않는다.
   - 설명 변수 후보나 단지 비교 참고 자료로만 사용한다.

4. 원천 파일은 형식이 제각각이다.
   - 일부는 `utf-8-sig + |` 구분자다.
   - 일부는 `cp949 + ,` 구분자다.
   - 서울 정비사업 자료는 `.xls`다.
   - 전처리 없이 바로 읽는 것을 전제로 하지 않는다.

## 추천 조인 순서

1. `단지고유번호` 또는 공공 단지 식별자가 있으면 그 키를 최우선 사용
2. 없으면 `도로명주소` 정규화 후 DuckDB `좌표`와 조인
3. 그래도 안 붙으면 `지번주소 + 단지명 정규화` 기준 보수적 조인
4. 외부 가공본은 최종 fallback 또는 품질 비교용으로 사용

## 추천 변수 사용 방식

### 전국 공통
- 연식
- 세대수
- 동수
- 가격 수준
- 거래량
- 급지/상대서열
- 전세가율, 월세화
- 정책/공급 변수

### 수도권 심화
- 역세권 지수
- 학교/학원 접근성
- 상권 접근성
- 종합병원/대규모점포 접근성
- 공원/한강 접근성
- CBD 접근성
- 정비사업 구역/근접성

### 서울 해석 시 특히 주의할 점
- 노후 구축은 단순 감가상각으로만 읽지 않는다.
- 재건축 기대, 토지 희소성, 대체 불가능한 입지 독점성 때문에 `연식이 오래될수록 오히려 강한 구간`이 존재할 수 있다.
- 따라서 서울/핵심 수도권에서는 `연식`을 반드시 `정비사업`, `한강`, `CBD`, `급지`, `생활권`, `가격수준`과 함께 해석한다.

## 현재 확인된 한계

- 수도권 심화용 위치변수는 전국 단위로 확장돼 있지 않다.
- 일부 파일은 원천 형식이 달라 재가공이 필요하다.
- 정비사업 자료는 지역별 형식이 달라 통합 정규화 테이블을 아직 만들지 않았다.
- 현재 프로젝트의 DuckDB에는 좌표 테이블이 있지만, 단지 수준 표준 키 체계는 추가 정리가 필요하다.
