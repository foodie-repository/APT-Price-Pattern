# 00. 전처리 및 매칭 요약

기준일: `2026-03-12`

## 1. 목적

`01_기준문서/02_실행가이드/execution_guide.md`의 `2. 전처리 및 분석 테이블 준비` 단계에서,
전국 공통 상품성 변수와 수도권 심화 입지 변수를 실제 DuckDB 뷰로 분리하고,
조인 규칙과 매칭률을 재현 가능한 형태로 고정했다.

- `단지_메타정보.csv`는 정식 단지 마스터가 아니라 보조 메타로만 사용한다.

## 3. 조인 규칙

### 3.1 전국 공통 상품성

우선순위:
1. `지번주소 + 단지명_공백제거` 유일매칭
2. `지번주소` 단독 유일매칭
3. `지번주소 + 사용승인연도` 유일매칭
4. 그 외는 미매칭

설명:
- `공동주택_전국`은 단지명이 거래 데이터와 다르게 적재된 경우가 많아서, 이름만 고집하면 매칭률이 크게 낮아진다.
- 동일 `지번주소`에 아파트 단지가 하나뿐인 경우는 주소 단독 유일매칭을 허용했다.
- 동일 주소에 복수 단지가 있는 경우만 `건축년도 ~= 사용승인연도` 보조 조건을 제한적으로 사용했다.

### 3.2 서울 내부 좌표/입지

우선순위:
1. `공동주택_기본정보 + 좌표` 내부 매칭
2. 실패 시 수도권 외부 심화 데이터로 fallback

정규화:
- `공동주택_기본정보.법정동주소`에는 단지명이 뒤에 붙어 있어,
  `법정동주소 - 단지명` 규칙으로 `지번주소`를 재생성했다.
- 이 정규화 없이는 서울 내부 보강이 거의 붙지 않는다.

### 3.3 수도권 외부 심화 입지

조인키:
- `지번주소 = 시군구 + 번지`
- `단지명_공백제거`

설명:
- 외부 위치변수 CSV의 `주소` 컬럼은 단지명까지 포함하므로 직접 주소 문자열 매칭은 사용하지 않았다.
- `시군구`, `번지`, `단지명`을 분리해 만든 키로 매칭했다.

### 5.3 수도권 호가 데이터 매칭 규칙

호가 원천:
- DuckDB `네이버부동산_단지`
- DuckDB `네이버부동산_매물`

현재 스냅샷 규모:
- 매매 최신 스냅샷 `74,579건`
- 전세 최신 스냅샷 `15,200건`

해석 원칙:
- 호가는 현재시장 체감과 매물 압력 확인용 보조지표로만 사용한다.
- 호가 비교 기준 실거래는 반드시 `직거래`, `해제 건`이 제외된 `v_sale_clean` 기반으로만 잡는다.
- 단지명은 소스 간 차이가 커서 `지번주소`를 우선 축으로 사용하고, 단지명은 보조 규칙으로만 쓴다.

우선순위:
1. `지번주소 + 정확단지명`
2. `지번주소 + 조인단지명`
3. `지번주소 유일`
4. 나머지는 미매칭

수도권 매매 실거래 기준 audit:
- `주소+정확단지명`: `642,835건`
- `주소+조인단지명`: `33,874건`
- `주소유일`: `872,971건`
- `미매칭`: `3,319,962건`

수도권 임차 실거래 기준 audit:
- `주소+정확단지명`: `743,699건`
- `주소+조인단지명`: `66,503건`
- `주소유일`: `879,264건`
- `미매칭`: `5,521,690건`

해석:
- 주소 유일매칭 비중이 크므로, 수도권 호가-실거래 연결은 단지명 완전일치보다 `주소 축`이 더 중요하다.
- 단지명 차이의 대표 예시는 `주공4` vs `주공4단지`, `당정마을엘지` vs `당정마을LG2차`, `이즈라이프` vs `이즈라이프(주상복합)` 유형이다.
- 따라서 단지명 차이만으로 자동 미매칭 처리하면 커버리지가 크게 떨어지고, 반대로 주소만 무조건 허용하면 오매칭 위험이 커진다. 이번 규칙은 둘 사이의 보수적 절충안이다.

급매 판정 규칙:
- `급매` 태그는 그대로 신뢰하지 않는다.
- 기준가격은 `최근 6개월 중앙 실거래가 -> 최근 실거래가 -> 최근 12개월 중앙 실거래가` 순으로 사용한다.
- 면적이 맞는 실거래가 있으면 면적 기준을 우선 쓰고, 없으면 동일 단지 `전용면적_구분` 기준으로 fallback 한다.
- 기준가격 대비 할인율 `10% 이상`: `강한가격기준급매`
- 기준가격 대비 할인율 `5% 이상`: `가격기준급매`
- 급매 태그는 있으나 할인폭이 작으면 `태그만급매`

조건부 가격 신호 기준:
- 비교집단 `5개 단지 이상`
- 단지 거래 `5건 이상`
- peer 대비 `-10% 이하`: `가격반영부족가능`
- peer 대비 `+10% 이상`: `과대반영가능`

## 7. 한계

- 전국 공통 변수는 아직 `세대수`, `동수`, `사용승인일` 중심이다.
- 전국 단위 `브랜드`, `주차대수`, `최고층`, `복도유형`, `난방방식`은 확보되지 않았다.
- 수도권 외부 심화 변수는 범위가 서울/경기/인천에 한정된다.
- `단지_메타정보.csv`는 보조 메타이므로 조건부 설명용으로만 다뤄야 한다.
- 재건축/재개발 전후 단지명 이력, 단지 통합/분할 이력은 아직 표준화되지 않았다.
