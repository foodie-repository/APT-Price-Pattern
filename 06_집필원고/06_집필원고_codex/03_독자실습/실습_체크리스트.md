# 독자 실습 체크리스트

## 목표

독자가 책을 읽기만 하지 않고, 실제로 자기 환경에서 분석 체계를 재현하도록 돕는 체크리스트다.

## 실습 0. 환경 준비

- [ ] 저장소 구조를 이해했다.
- [ ] DuckDB 파일 위치를 확인했다.
- [ ] 실습에 사용할 최소 데이터 범위를 정했다.
- [ ] 기준 문서와 실행 가이드 위치를 확인했다.

## 실습 1. 데이터 구조 파악

- [ ] 매매, 전세, 월세, 정책, 공급 데이터의 역할을 구분했다.
- [ ] 지역 키와 단지 키를 확인했다.
- [ ] 계약년월, 면적, 가격, 연식 관련 컬럼을 확인했다.

## 실습 2. 전처리

- [ ] 직거래/해제 제외 규칙을 이해했다.
- [ ] 조인 키를 정리했다.
- [ ] 면적 구분과 연식 구분을 만들었다.

## 실습 3. 해석

- [ ] 장기 역사와 현재 국면을 분리해서 설명할 수 있다.
- [ ] 전세와 월세를 섞지 않고 설명할 수 있다.
- [ ] 정책과 공급을 가격 옆에 놓고 읽을 수 있다.

## 실습 4. 후보 선별

- [ ] 후보를 `우선 검토 / 판단 보류 / 즉시 제외`로 나눌 수 있다.
- [ ] 가격 반영이 덜 된 신호와 가격이 앞서간 신호를 동시에 볼 수 있다.
- [ ] 임차 시장의 지지력, 입주예정 물량, 미분양을 같이 볼 수 있다.

## 실습 5. 노트북 시각화 검증

- [ ] 보고서 핵심 수치를 차트로 다시 확인할 포인트를 정했다.
- [ ] 장기 시계열, 급지 비교, 후보 검증 차트의 역할을 구분했다.
- [ ] 노트북은 `분석 보조/검증용`, 보고서는 `설명/의사결정용`이라는 차이를 이해했다.
- [ ] 향후 노트북 산출물 저장 위치를 `04_결과/04_노트북/`로 연결할 준비를 했다.

## 실습 6. 시스템화

- [ ] 내 질문을 시스템 질문으로 바꿀 수 있다.
- [ ] 월별 업데이트 루틴을 설계했다.
- [ ] 내 판단 로그를 남길 양식을 만들었다.
